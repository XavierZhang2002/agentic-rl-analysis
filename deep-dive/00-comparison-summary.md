## 2. 关键概念对比

### 2.1 训练阶段设计

| 工作 | 训练阶段 | 关键创新 |
|------|---------|---------|
| **Tongyi** | Pre → **Mid-training** → RL | 显式定义Mid-training阶段，专门培养agentic偏好 |
| **GLM-5** | Pre → **Mid-training** → SFT → RL | 同样采用Mid-training，强调数据混合策略 |
| **ABE** | 不指定 | 专注于环境，与训练阶段解耦 |
| **GEM** | 不指定 | 专注于数据合成，可与任何训练框架结合 |

**共识**：Mid-training已成为Agentic RL的标配

### 2.2 环境建模

| 工作 | 环境类型 | 特点 |
|------|---------|------|
| **Tongyi** | Prior / Simulated / Real-world | 系统性三级划分 |
| **GLM-5** | 不重点讨论 | 侧重推理模式 |
| **ABE** | Simulated（自动化构建） | 专注于Simulated环境的生成 |
| **GEM** | 不直接涉及 | 通过数据模拟环境 |

**互补性**：
- Tongyi提供框架
- ABE提供Prior→Simulated的构建工具

### 2.3 数据合成

| 工作 | 数据源 | 合成方法 |
|------|--------|---------|
| **Tongyi** | 知识图谱 | Graph Construction + Uncertainty Injection |
| **GLM-5** | 未重点讨论 | 依赖Mid-training的数据混合 |
| **ABE** | 自动生成 | Scenario Decomposition + Complexity Scaling |
| **GEM** | 文本 | Text Filtering → Extraction → Generation → Refinement |

**多样性**：
- Tongyi：结构化数据（图谱）
- GEM：非结构化数据（文本）
- ABE：程序化生成

## 7. 关键洞察总结

### 7.1 关于Agentic RL的本质

**四篇工作的共识**：
1. **数据质量 > 模型架构**：合成高质量数据是关键
2. **环境工程是基础**：没有好的训练环境，再好的算法也没用
3. **训练阶段很重要**：Mid-training填补了Pre-training和RL之间的gap
4. **推理模式需要灵活性**：不同任务需要不同的思考深度

### 7.2 关于工程实践

**可落地的技术**：
- ✅ Mid-training：已有成功案例（Tongyi, GLM-5）
- ✅ 自动化环境构建：ABE提供了完整方案
- ✅ 数据合成：GEM证明了可行性
- ⚠️ 三级环境：Simulated→Real的gap仍需解决

### 7.3 关于研究方向

**值得关注的问题**：
1. **Sim-to-real gap如何量化？**
2. **如何判断Mid-training的收敛？**
3. **Thinking Format如何选择最优策略？**
4. **合成数据的质量如何自动评估？**
